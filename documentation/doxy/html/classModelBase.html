ModelBase::addfield (   $name,
  $object 
)
protected

Adds a field to the model. This is intended to be called in the constructor of the classes that inherit this class in order to register the fields and relational functionality with the class and the data access layer. Adding fields dynamically is not supported, so adding columns to the database after the model has been registered with the data access layer the first time needs to be done manually.

Parameters
$name- the name of the field to add
$object- an instance of a Model field.
Exceptions
Exception- throws an exception for unsupported field types

ModelBase::as_array ( )

Returns the contained data of the object as an associative array stripped of the field types. This is used when saving and deleting an object, as the field types have been validated by the class already and the types are already configured in the database.

ModelBase::delete ( )

Deletes a model instance from the database via the data access layer. It is recommended that the model be registered with the data access layer beforehand in order to make sure the necessary database structures exist.

Exceptions
Exception- throws an exception if the model does not contain fields or data.
static ModelBase::fromArray (   $array)
static

Creates a new instance of the model and populates it from an array. The array must contain ONLY the data that the model is expected to contain, or the model instance will throw exceptions. This method is used by the data access layer to create model instances from an associative array returned by the database layer.

Parameters
$array- an associative array (string=>string) of data for the model. This array does not contain fields or types, as the model is initialized first and those are set in the model's own constructor.
Returns
$instance - a populated model instance
